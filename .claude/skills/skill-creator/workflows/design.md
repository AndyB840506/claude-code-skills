# Step 2 — Design the Skill

Plan before writing.

## 5 Design Elements

1. **Input** — What you ask the user for (data, context, preferences)
2. **Process** — Steps it follows (in order)
3. **Tools** — What it uses (WebFetch, Bash, Read, Write, native tools)
4. **Output** — What it generates and format (file, HTML, text)
5. **UX** — How it feels (messages, conversational flow)

## Example

| Element | Value |
|---|---|
| Input | CSV of products (user uploads file) |
| Process | Read CSV → Parse data → Generate HTML |
| Tools | Read (CSV file), Write (save HTML) |
| Output | HTML file with product cards |
| UX | Friendly messages, progress indicator |
