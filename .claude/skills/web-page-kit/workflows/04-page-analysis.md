# Workflow 04 — Page Analysis: Audit Existing Pages

Analyze an existing podcast webpage or any competitor site for SEO, design, and performance issues. Provides detailed audit report with actionable improvements.

## Step 0: Optional Workflow

This is **optional** — only needed if analyzing an existing page.

---

## Step 1: Target Page

Ask:

1. **Page URL** — Which page to analyze? (paste full URL)
2. **Purpose** — Why analyze? (improve own page, learn from competitor, diagnostic)

**ACTION:** Fetch the page using WebFetch.

---

## Step 2: SEO Analysis

Analyze and report on:

1. **Page Title** — Is it optimized? (50-60 chars ideal)
2. **Meta Description** — Quality and length (155-160 ideal)
3. **Headers (H1, H2, H3)** — Structure and keyword usage
4. **Keyword Density** — Is primary keyword optimized? (1-2%)
5. **Alt Text** — Images have descriptive alt tags?
6. **Internal Links** — Navigation and internal linking?
7. **External Links** — Quality outbound links?
8. **Mobile Responsiveness** — Passes mobile test?
9. **Page Speed** — Estimated load time (from HTML size)
10. **Structured Data** — Schema.org present?
11. **Social Tags** — OG and Twitter Card tags present?
12. **Security** — HTTPS? No mixed content?

---

## Step 3: Design & UX Review

Ask & analyze:

1. **Color Scheme** — Primary colors used? Contrast OK?
2. **Typography** — Readable fonts? Proper hierarchy?
3. **Layout** — Clear navigation? CTA visible?
4. **Images** — Quality, optimization, relevance
5. **Mobile Layout** — How does it respond on mobile?

---

## Step 4: Performance Metrics

Estimate & report:

1. **Estimated Load Time** — Based on file sizes
2. **Image Optimization** — Are images compressed?
3. **CSS/JS** — Are files minified?
4. **Caching Strategy** — Browser caching present?

---

## Step 5: Generate Audit Report

**ACTION:** Create structured audit report:

```
=== SEO ANALYSIS ===
✅ Title Tag: [score] — [recommendation]
⚠️ Meta Description: [score] — [issue & fix]
✅ Keywords: Well-optimized
❌ Schema.org: Missing — [add this structured data]

=== DESIGN ===
✅ Layout: Professional, clear hierarchy
⚠️ Mobile: Responsive but could optimize landscape

=== PERFORMANCE ===
Estimated Load: [X] seconds
⚠️ Image Optimization: [recommendation]

=== STRENGTHS ===
- [What's working well]
- [Design strength]

=== IMPROVEMENTS (Ranked by Impact) ===
HIGH IMPACT:
1. Add schema.org Podcast data
2. Optimize meta description
3. Add alt text to images

MEDIUM IMPACT:
4. Improve H1 keyword usage
5. Add internal linking

LOW IMPACT:
6. Minor color contrast tweak
```

---

## Step 6: Recommendations

Provide:
- ✅ Detailed audit report (PDF-style or markdown)
- ✅ Side-by-side comparison (current vs. recommended)
- ✅ Code snippets for fixes (ready to copy)
- ✅ Priority checklist

---

**Page analysis complete!** Use these recommendations to improve your site or learn from competitors.
